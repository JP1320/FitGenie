# Data Model: Fit Card

## Purpose

The Fit Card is a structured, shareable document auto-generated at booking time. It consolidates the user's measurements, body type, style preferences, outfit selection, and tailor notes into a single artefact that is shared with the tailor/designer.

## Field Definitions

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | UUID | Yes | Primary key |
| `user_id` | UUID | Yes | Card owner |
| `booking_id` | UUID | Yes | Associated booking |
| `tailor_id` | UUID | Yes | Recipient tailor/designer |
| `measurement_id` | UUID | Yes | Snapshot of measurements at time of booking |
| `preference_id` | UUID | Yes | Snapshot of preferences at time of booking |
| `version` | Integer | Yes | Incremented if card is regenerated |
| `user_summary` | Object | Yes | Safe subset of user data for tailor view |
| `measurements` | Object | Yes | Copied from measurement record at booking time |
| `style_preferences` | Object | Yes | Copied from preference record at booking time |
| `outfit_reference` | Object | Yes | The selected outfit from recommendation |
| `tailor_notes` | String | No | User-authored notes to tailor; max 500 chars |
| `ai_notes` | String | No | AI-generated stitching guidance for the tailor; max 300 chars |
| `share_token` | String | Yes | URL-safe token for sharing the card as a link |
| `share_expires_at` | Timestamp | No | Optional expiry for shared link; null = no expiry |
| `is_shared` | Boolean | Yes | True once the card has been sent to the tailor |

## Sharing Mechanism

The Fit Card can be shared in two ways:

1. **In-app:** Tailor receives a notification with a link to view the card within the FitGenie platform.
2. **Public link:** A URL using `share_token` allows viewing without login (read-only, anonymised). Format: `https://fitgenie.app/fit-card/{share_token}`

The share link renders:
- Measurements table
- Body type, style preferences, outfit reference
- Tailor notes and AI notes
- It does NOT render: age, exact coordinates, auth data, or weight.

---

## Versioning

When a user updates measurements or preferences after initial card generation (e.g., corrects a measurement), a new version of the Fit Card is issued. The tailor is notified of the update. Previous versions are retained for audit.

---

## AI Notes Generation

The `ai_notes` field is generated by the AI layer using the Fit Card Generation prompt. It provides the tailor with:

- Suggested stitching adjustments for the user's body proportions.
- Key areas to pay attention to (e.g., "shoulder width is slightly wider than standard; adjust armhole accordingly").
- Fit type preference reminders (e.g., "user prefers relaxed fit; add 2–3 cm ease at chest").
