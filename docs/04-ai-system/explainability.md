# Explainability

## Overview

FitGenie's explainability layer ensures that every AI-generated recommendation is accompanied by a clear, human-readable rationale that the user can understand and act on. This applies to outfit recommendations, size suggestions, and fit analysis results.

---

## Why Explainability Matters

1. **Trust:** Users are more likely to act on a recommendation they understand.
2. **Body positivity:** Explanations must frame recommendations positively, focusing on garment properties rather than body "flaws".
3. **Accountability:** The explainability layer creates a traceable audit trail of why a recommendation was made.
4. **Correction:** If a user disagrees with a recommendation, they can identify which input was misaligned and correct it.

---

## Explanation Types

### Type 1: Recommendation Explanation

Attached to each item in the recommendation response. Generated by the AI language model.

**Structure:**

```json
{
  "summary": "string (≤ 150 chars)",
  "body_type_match": "string (1–2 sentences)",
  "style_match": "string (1 sentence)",
  "fabric_match": "string (1 sentence) | null",
  "occasion_match": "string (1 sentence) | null"
}
```

**Example:**

```json
{
  "summary": "This A-line kurta complements your proportions and matches your festive style.",
  "body_type_match": "The flared hem and fitted bodice create a balanced silhouette for your body type.",
  "style_match": "Matches your Ethnic style preference.",
  "fabric_match": "Lightweight cotton — your preferred fabric — keeps it comfortable.",
  "occasion_match": "Perfectly suited for festive occasions like weddings."
}
```

---

### Type 2: Size Rationale

A concise explanation of why a specific size label was chosen. Included in the size recommendation object.

**Example:**
> "Your chest measurement of 92 cm places you comfortably in size M for this garment. The regular ease in this design means the fit will be relaxed as you prefer."

**Rules:**
- Must reference a specific measurement (not generic).
- Must not mention weight.
- Must not state comparisons to average or typical users.

---

### Type 3: Fit Analysis Explanation

Included in the detailed fit analysis response (triggered on demand or for Fit Card AI notes).

**Example:**
> "The chest and waist fit well within your preferred regular range. The shoulder measurement is slightly narrower than ideal — a minor adjustment of 1 cm at each shoulder seam will improve the fit."

---

### Type 4: Confidence Explanation

Static, pre-defined copy surfaced when the user taps the confidence badge (see `confidence-scoring.md`). Not AI-generated.

---

## Language Rules for All Explanations

These rules are enforced via prompt guardrails (see `prompts/prompt-guardrails.md`):

| Rule | Example of Violation | Compliant Alternative |
|---|---|---|
| Never reference body weight | "Your weight means you should choose…" | Remove weight references entirely |
| Never use negative size language | "You are a large size" | "Your measurements suit size L" |
| Never compare to others | "Most people your height choose…" | Focus on the individual's measurements only |
| Always frame as garment properties | "You're too wide for this design" | "This design is cut narrow; a size up gives more ease" |
| Avoid filler hedging | "We think maybe this might possibly suit you" | Be precise: "This suits you because…" |
| Maintain inclusive language | Gender-assuming language | Use gender-neutral language or match user's stated gender preference |

## Explanation Generation Flow

```
[Recommendation/Fit Data]
        ↓
[Build prompt: measurements + garment specs + user context]
        ↓
[Call AI LLM with explanation prompt template]
        ↓
[Validate output against guardrails]
        ↓
[If validation fails: retry once, then use fallback template]
        ↓
[Return explanation fields]
```

---

## Fallback Explanation Templates

Used when AI generation fails or produces guardrail violations:

- **Summary fallback:** `"This item matches your size and style preferences."`
- **Body type match fallback:** `"This garment's silhouette is suited to your measurements."`
- **Style match fallback:** `"This aligns with your selected style preferences."`

Fallback explanations are flagged internally (`explanation_source: "fallback"`) for monitoring.

---

## Non-Goals

- The explainability layer does not expose AI model weights, training data, or internal scoring details to users.
- It does not generate explanations for catalogue items the user has not been recommended.
- It does not provide medical or health-related commentary on measurements.
