# Confidence Scoring

## Overview

The Confidence Score quantifies how certain the FitGenie system is that a recommended size label will fit the user correctly. It is presented to the user as a numeric score and a colour-coded band, and it informs recommendation ranking.

---

## Confidence Score Definition

The confidence score is a float value between `0.0` and `1.0`:

- `1.0` = certainty that the size label is correct for this user.
- `0.0` = no basis for a size determination.

Scores are never displayed as raw floats to end users; they are always translated into a band and badge.

---

## Score Components

The overall confidence score is a weighted composite of four sub-scores:

| Sub-Score | Weight | Description |
|---|---|---|
| `measurement_completeness` | 0.30 | How many relevant measurements are available |
| `fit_analysis_score` | 0.40 | Output of the fit analysis algorithm (ease within tolerance) |
| `scan_quality` | 0.20 | Confidence from the AI scanner (or 1.0 for confirmed manual input) |
| `size_boundary_clarity` | 0.10 | Whether measurements fall cleanly within a size, not near a boundary |

```
confidence_score = (measurement_completeness × 0.30)
                 + (fit_analysis_score × 0.40)
                 + (scan_quality × 0.20)
                 + (size_boundary_clarity × 0.10)
```

---

## Sub-Score Calculation

### 1. Measurement Completeness

Based on which measurements are present for the garment type:

| Garment Type | Required Measurements | Optional |
|---|---|---|
| Top / Shirt | chest, shoulder | sleeve, neck |
| Bottom / Trouser | waist, hips | inseam, thigh |
| Dress / Set | chest, waist, hips | shoulder, sleeve |

Score:
```
measurement_completeness = (present_required / total_required) × 0.7
                         + (present_optional / total_optional) × 0.3
```

### 2. Fit Analysis Score

Taken directly from the `fit_score` output of the fit analysis algorithm (see `fit-analysis-logic.md`).

### 3. Scan Quality

- AI scan sessions: taken from the scanner model's per-session confidence output (0.0–1.0).
- Manual input confirmed by user: `1.0`.
- Manual input not confirmed (fields pre-filled but not reviewed): `0.85`.
- Partial manual input (some fields missing): scaled proportionally.

### 4. Size Boundary Clarity

Measures how far the user's primary measurement is from the boundary between two sizes:

```
boundary_distance_pct = min(distance_to_lower_boundary, distance_to_upper_boundary) / size_step_width

size_boundary_clarity = min(1.0, boundary_distance_pct / 0.3)
```

If a user's chest measures exactly at the M/L boundary (e.g., 96 cm when M = 92–96 and L = 96–100), `size_boundary_clarity = 0.0`, reflecting that either size is plausible.

## Confidence Band Mapping

| Score Range | Band | UI Badge | Colour |
|---|---|---|---|
| 0.80 – 1.0 | `high` | "Great Match" | Green |
| 0.60 – 0.79 | `medium` | "Good Match" | Amber |
| 0.00 – 0.59 | `low` | "Possible Match" | Grey |

---

## User-Facing Explanation of Confidence

When a user taps the confidence badge, they see:

**High (≥ 0.8):**
> "We're highly confident this is the right size for you, based on your measurements."

**Medium (0.6–0.79):**
> "We think this size is a good fit. Your measurements are near the size boundary — you may want to confirm with the tailor."

**Low (< 0.6):**
> "We've suggested a size, but we'd recommend confirming measurements before stitching."

These messages are fixed copy — they are NOT generated by the AI language model. This ensures consistency and avoids confidence over-claiming.

---

## Confidence for Manual Input vs AI Scan

| Source | Max Achievable Score | Notes |
|---|---|---|
| AI scan (high quality) | 1.0 | Scanner confidence ≥ 0.9 |
| AI scan (medium quality) | 0.85 | Scanner confidence 0.6–0.89 |
| Manual (all fields confirmed) | 0.95 | Scan quality = 1.0; slight reduction for size boundary risk |
| Manual (partial fields) | 0.70 | Measurement completeness penalty |
| Guest (no profile data) | 0.40 | Limited data; fallback size suggested |

---

## Non-Goals

- Confidence scores are not shared with tailors (Fit Card shows measurements directly; tailor makes their own judgment).
- Confidence scores are never used to discourage users from placing orders — low scores generate a guidance note, not a block.
